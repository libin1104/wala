# 触发示例

## 一、创建类请求

这些说法都应触发 skill，并映射到：

- `scripts/manage_schedule_tasks.py context`
- 由当前会话模型生成任务 JSON
- `scripts/manage_schedule_tasks.py create-from-json`

典型表达：

- “创建定时任务，每天早上九点提醒我整理日报”
- “每 30 分钟检查一次构建状态，结果发邮件给我”
- “帮我加一个循环任务，每小时汇总一次错误日志”
- “明天下午三点提醒我一次，发邮件告诉我去开会”
- “2026-03-20 18:00 给我发一封项目里程碑提醒”
- “每天晚上 8 点给我发工作总结提醒”
- “定期发我一封项目进展汇总邮件”
- “自动帮我盯一下线上错误，每小时发一次摘要”
- “以后每天早上提醒我看 Jira”
- “帮我设一个任务，每隔 10 分钟检查一下接口状态”

口语化但也应该触发：

- “帮我盯一下这个事情，每小时跟我汇报一次”
- “之后固定提醒我发日报”
- “给我搞个循环提醒，晚上八点发我”
- “想让它每天早上自动给我一封邮件”
- “弄个定时的，半小时检查一次就行”
- “明天三点到点叫我一下”
- “下周一早上九点给我来一封提醒邮件”

## 二、查看类请求

这些说法都应触发 skill，并映射到：

- `scripts/manage_schedule_tasks.py list`

典型表达：

- “看看当前有哪些定时任务”
- “现在有哪些循环任务”
- “列一下我现在的提醒任务”
- “帮我看下任务列表”
- “我都配了哪些自动邮件”

## 三、修改类请求

这些说法都应触发 skill，并映射到：

- `scripts/manage_schedule_tasks.py context`
- 由当前会话模型生成替换任务 JSON
- `scripts/manage_schedule_tasks.py update-from-json "<selector>"`

典型表达：

- “把每周汇总改成每天晚上八点”
- “把那个日报任务改成每小时执行一次”
- “把错误日志汇总改成每天早上发”
- “把那个提醒邮件内容改得更简洁一点”
- “把刚才那个任务改成每 15 分钟检查一次”

如果用户没有明确 selector，但上下文明显在接着上一个任务说“改一下”，先 `list` 或结合刚完成的任务确认对象。

## 四、启停类请求

这些说法都应触发 skill：

- 启用：`scripts/manage_schedule_tasks.py enable "<selector>"`
- 停用：`scripts/manage_schedule_tasks.py disable "<selector>"`

典型表达：

- “停用那个日报任务”
- “暂停刚才那个日志汇总任务”
- “先别发那个提醒了”
- “恢复刚才暂停的日志汇总任务”
- “重新启用那个每天九点的任务”

## 五、删除类请求

这些说法都应触发 skill，并映射到：

- `scripts/manage_schedule_tasks.py delete "<selector>"`

典型表达：

- “删除那个测试任务”
- “把刚才建的提醒删掉”
- “不要这个循环任务了”
- “把那个每天九点的邮件任务移除”

## 六、不该触发的例子

这些都不是“创建或管理循环任务”，不该走这个 skill：

- “帮我解释这个报错”
- “把这段文案润色一下”
- “现在给我发一封邮件”
- “帮我总结这段对话”

原因：

- 这些是普通问答、即时动作或内容处理，不是“创建或管理定时任务”

## 七、需要先澄清的例子

这些请求要先确认，再决定是否用这个 skill：

- “之后提醒我一下”
- “晚点记得告诉我”
- “后面帮我关注这个”
- “帮我定时看一下”
- “找个时间提醒我一次”

澄清方向：

- 这是一次性提醒，还是循环提醒？
- 如果是一次性提醒，绝对时间是什么？
- 如果是循环提醒，频率是“每隔多少分钟”还是“每天几点”？

## 八、歧义处理

如果一个 selector 匹配到多个任务，先执行 `list`，再让用户明确任务 ID 或精确名称。
